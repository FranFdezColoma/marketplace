# Naming Conventions for Dynamics 365 & Power Platform

> **Single source of truth** for naming every artifact type in D365 / Power Platform projects.
> Replace `{prefix}_` with your publisher prefix (e.g. `contoso_`, `acme_`).

---

## 1. Dataverse Tables (Entities)

| Rule | Detail |
|---|---|
| **Format** | `{prefix}_{entity_name}` |
| **Casing** | `snake_case`, **singular** |
| **Standard tables** | Use as-is â€” `account`, `contact`, `lead`, `opportunity`, `incident` |
| **Activity tables** | `{prefix}_{name}` registered as an activity type |

**Examples**

```
{prefix}_project
{prefix}_invoice_line
{prefix}_time_entry
{prefix}_approval_request
```

> **Do NOT** pluralise table names. The platform handles pluralisation for display automatically.

---

## 2. Dataverse Columns (Fields / Attributes)

| Rule | Detail |
|---|---|
| **Format** | `{prefix}_{column_name}` â€” `snake_case` |
| **Lookup** | `{prefix}_{name}id` â€” suffix with `id` |
| **Boolean** | `{prefix}_is{name}` or `{prefix}_has{name}` |
| **Date** | `{prefix}_{name}_date` or `{prefix}_{name}on` (e.g. `{prefix}_startedon`) |
| **Currency** | `{prefix}_{name}_amount` (e.g. `{prefix}_total_amount`) |
| **Status / State** | Use OOB `statecode` / `statuscode`; extend `statuscode` for custom statuses |
| **Choice / OptionSet** | `{prefix}_{name}` with integer values starting at the publisher's option value prefix |

**Examples**

```
{prefix}_name                  -- String
{prefix}_projectid             -- Lookup to {prefix}_project
{prefix}_isactive              -- Boolean (Two Option)
{prefix}_hasdependencies       -- Boolean
{prefix}_startedon             -- DateTime
{prefix}_total_amount          -- Currency
{prefix}_priority              -- Choice
```

## 3. Choices (Option Sets / PickLists)

| Scope | Rule |
|---|---|
| **Global Choice** | `{prefix}_{choicename}` (e.g. `{prefix}_projectstatus`) |
| **Local Choice** | Bound to the column â€” no separate schema name needed |
| **Values** | Start at publisher's option value prefix (e.g. `100000000`) |
| **Labels** | PascalCase or Title Case (`InProgress` or `In Progress`) |

**Example â€” Global Choice `{prefix}_projectstatus`**

| Value | Label |
|---|---|
| 100000000 | Draft |
| 100000001 | In Progress |
| 100000002 | On Hold |
| 100000003 | Completed |
| 100000004 | Cancelled |

## 4. Relationships

| Type | Format | Example |
|---|---|---|
| **1:N** | `{prefix}_{parenttable}_{childtable}` | `{prefix}_account_project` |
| **N:N** | `{prefix}_{table1}_{table2}` (alphabetical) | `{prefix}_project_resource` |

> Use the **singular** logical name of each table in the relationship name.

## 5. Solutions

| Artifact | Convention | Example |
|---|---|---|
| **Publisher** | PascalCase, no spaces | `Contoso` |
| **Solution unique name** | `{Publisher}{SolutionName}` | `ContosoSalesExtensions` |
| **Display name** | Human-readable with spaces | `Contoso Sales Extensions` |
| **Segmented** | `{Publisher}{Module}{Layer}` | `ContosoSalesEntities`, `ContosoSalesPlugins` |
| **Patch** | `{SolutionName}_Patch_{GUID}` | Auto-generated by platform |

### Recommended Segment Layers

```
{Publisher}Core           -- Tables, columns, relationships, global choices
{Publisher}Plugins        -- Plugin assemblies and steps
{Publisher}Flows          -- Cloud Flows, Business Rules
{Publisher}UI             -- Forms, views, dashboards, sitemap
{Publisher}Security       -- Security roles
{Publisher}WebResources   -- JavaScript, HTML, CSS, images
{Publisher}PCF            -- PCF components
```

## 6. Plugins (C#)

| Artifact | Convention | Example |
|---|---|---|
| **Namespace** | `{Company}.{Project}.Plugins` | `Contoso.Sales.Plugins` |
| **Class** | `{Entity}{Action}{Stage}Plugin` | `AccountValidatePrePlugin` |
| **Project** | `{Company}.{Project}.Plugins` | `Contoso.Sales.Plugins` |
| **Test project** | `{Company}.{Project}.Plugins.Tests` | `Contoso.Sales.Plugins.Tests` |
| **Assembly** | `{Company}.{Project}.Plugins.dll` | `Contoso.Sales.Plugins.dll` |

### Common Action / Stage Combinations

```
AccountValidatePrePlugin          -- Pre-Validation (Stage 10)
InvoiceLineCalculatePrePlugin     -- Pre-Operation  (Stage 20)
OpportunityCreatePostPlugin       -- Post-Operation (Stage 40)
CaseEscalatePostAsyncPlugin       -- Post-Operation Async
```

## 16. Environment Variables

| Artifact | Convention | Example |
|---|---|---|
| **Schema name** | `{prefix}_{Category}_{Name}` | `{prefix}_Integration_ApiKey` |

### Supported Types

- `String` â€” URLs, keys, identifiers
- `Number` â€” Thresholds, limits
- `Boolean` â€” Feature flags
- `JSON` â€” Complex configuration objects
- `Data Source` â€” Connection strings

**Examples**

```
{prefix}_Integration_ApiBaseUrl       -- String
{prefix}_Feature_EnableAuditLog       -- Boolean
{prefix}_Batch_MaxRecordCount         -- Number
{prefix}_Mapping_FieldConfig          -- JSON
{prefix}_Reporting_DataSource         -- Data Source
```

## 17. Git Conventions

### Branches

```
Format:  {type}/{ticket-id}-{short-description}

feat/PROJ-123-account-validation
fix/PROJ-456-duplicate-detection
docs/PROJ-789-update-alm-guide
refactor/PROJ-101-plugin-base-class
chore/PROJ-202-update-dependencies
```

### Commits

Follow **Conventional Commits**:

```
feat: add account pre-validation plugin
fix: correct discount calculation rounding
docs: update naming conventions reference
refactor: extract base plugin class
test: add unit tests for invoice calculation
chore: upgrade SDK packages to latest
```

### Tags

Semantic Versioning: `v{major}.{minor}.{patch}`

```
v1.0.0
v1.1.0
v1.1.1
```

## 18. Test Naming

### C# (MSTest / xUnit)

```
Format: MethodName_StateUnderTest_ExpectedBehavior

CalculateDiscount_AmountAboveThreshold_ReturnsDiscountedPrice
ValidateAccount_MissingRequiredFields_ThrowsInvalidPluginExecutionException
CreateInvoiceLine_ValidInput_CreatesRecordSuccessfully
```

### JavaScript / TypeScript (Vitest / Jest)

```
Format: should {expected behavior} when {condition}

describe('Account.OnLoad', () => {
  it('should hide credit limit field when account type is Prospect')
  it('should set name as required when form type is Create')
  it('should show notification when balance exceeds threshold')
})
```
